## Claude で提供されるアプリの違い

なんとなく、凄さを伝えたところで、 **Claude** で使えるサービスを紹介（おさらい）しましょう。

- Claude では次は3つのツールが使えます
  - 1. Claude (Claude AI)
    - ChatGPTやGeminiみたいに、チャットができます
    - Claudeは、インタラクティブなグラフ（＝動くグラフ）をチャットの中に埋め込んでくれる機能もあります
  - 2. Claude Code
    - スーパーエンジニアのように、コーディングができます
    - AIエージェントが自律的にコードをかいて、パソコンの操作をすることもできます
    - 調べ物をしたり、ファイルの操作を自動化したり、Webページやアプリを作ったり、本格的なシステムを作ることもできます
  - Claude Cowork
    - まだ **research preview(＝研究)** 段階の機能
    - Claude Code と同じように、全自動でタスクを実行してくれる
    - タスクを指示したら、成果物を作るところまで実行して、結果を報告してくれます。つまり、 **※途中の承認は無しでやり切ってくれる！**

## Claude Code をターミナルで試してみましょう

### はじめにターミナルの操作を練習します

ターミナルを操作するのが初めてな方向けに、ターミナルの操作を少し練習してみましょう。（ちょっとエンジニアっぽい雰囲気を体験）

- ターミナルの基本的な使い方 (Claude Code もだいたい同じ)
 - マウス操作は ほぼ 使えません...
 - 1つ前のコマンドをもう一度使いたいときは **↑ キー** を押します
 - **(重要)** 処理を中断するときは **control + c**  ※ `Command` ではなく `Control` 
 - 終了するときは **赤い x ボタンでウィンドウを閉じる** でもOK
- ターミナルを使うときの注意点
 - **Enter** キーを押すときは慎重に。ちょっと怖いコマンドも実行できてしまうかも？
 - **半角/全角 どっち？** かを意識しましょう。
   - 通常は **半角モード** にします。日本語を入力するときだけ 全角モード にします。
- ターミナルの便利な使い方
 - (Macの場合)フォルダを **Dockのターミナルのアイコンにドラッグ＆ドロップ** すると、そのフォルダでターミナルを開けます。
 - 作業したいフォルダに移動する方法
   - `cd ` (シー ディー スペース) と入力した後に、 **ターミナルの画面にフォルダをドラッグ＆ドロップ** して **Enter** で決定 -> フォルダを移動できます

## もう少し本格的に使ってみましょう

とても便利な Claude Code ですが、正しく使わないと危険なこともあります。

例えば、セキュリティの設定をしないまま Claude Code に「なんでもOK！」で使ってしまうと、こんなことが起きるかもしれません。

- 🔑 **APIキーやパスワードが漏れちゃう**
  - 例えば「画像を生成したい！」と思って、Claude Code に言われるがまま APIキー（サービスを使うための秘密の鍵）を設定ファイル（`.env`）に保存したとします。セキュリティがガバガバだと、Claude Code がそのファイルの中身をうっかり外部に送信してしまう可能性があります。APIキーが漏れると、 **他人にあなたのアカウントを勝手に使われて、高額な請求が来る** なんてことも…！
- 🗑️ **大事なファイルを勝手に消されちゃう**
  - 「フォルダを整理して」とお願いしたら、Claude Code が「これは不要だな」と判断して、実は大切だったファイルを削除してしまう…なんてことも。
- ⚙️ **知らないソフトを勝手にインストールされちゃう**
  - Claude Code はコマンド（パソコンへの命令）を実行できます。「この機能を使うにはこのソフトが必要です、インストールしますか？」と聞かれて、よく分からないまま「OK」を押し続けていると、意図しないソフトがどんどん入ってしまうかもしれません。

> 😱 怖い！と思った方、安心してください。次の手順で **安全に使うための設定** をしていきます！

![](../assets/security-risks.png)

### 安心＆便利に Claude Code を使うための設定をしてみましょう

Claude Code を安心して便利に使えるように、少しだけ準備します

ガイドに沿って、 **順番にコピペ** しましょう

![](../assets/security-settings.png)

#### 🔧 設定①：画面のちらつきを無くす（No Flicker モード）

Claude Code は初期状態だと、文字が表示されるときに画面がチカチカすることがあります。
**No Flicker モード** を有効にすると、チャットアプリのように **スムーズに文字が表示** されるようになります！

**やり方はカンタン！** 次のコマンドをターミナルにコピペして **Enter** を押すだけです。

```bash
CLAUDE_CODE_NO_FLICKER=1 claude
```

> 💡 これは「ちらつきを無くすモードで Claude Code を起動してね」というコマンドです。
> いつも通り `claude` で起動するのと同じですが、今回だけ特別にちらつき防止がオンになります。

画面がスムーズに表示されるようになったら成功です 🎉

---

<details>
<summary>📖 （参考）毎回自動で No Flicker モードにしたい方はこちら</summary>

毎回 `CLAUDE_CODE_NO_FLICKER=1 claude` と入力するのは面倒ですよね。
設定を保存しておけば、次回から `claude` だけで No Flicker モードになります。

**ステップ1：設定を保存する**

```bash
echo 'export CLAUDE_CODE_NO_FLICKER=1' >> ~/.zshrc
```

> 「次回から No Flicker モードをオンにしてね」という設定を保存するコマンドです。
> ※ Mac をお使いの方はこのままでOKです。

**ステップ2：設定を今すぐ反映する**

```bash
source ~/.zshrc
```

> 「いま保存した設定を、今すぐ使えるようにしてね」というコマンドです。

**ステップ3：確認する**

```bash
echo $CLAUDE_CODE_NO_FLICKER
```

`1` と表示されたら成功です！これで `claude` だけで快適に使えます 🎉

> 詳しくは [No Flicker モードの設定ガイド](./no-flicker-setup.md) を参照してください

</details>

### Claude Code のセキュリティ対策

#### 🛡️ 対策①：作業フォルダを限定する

先ほど「新規フォルダを作って、その中で Claude Code を起動しましょう」とお伝えしました。
これは快適に使うためだけでなく、 **セキュリティ的にもとても大事** です。

Claude Code は、起動したフォルダの中にあるファイルを読んだり、書き換えたりできます。
もし **デスクトップ全体** や **パソコンの一番上のフォルダ（ホームフォルダ）** で起動してしまうと、関係のないファイルまで触れてしまいます。

> ✅ **専用のフォルダを作って、その中で起動する** ← これだけで安全性がグッと上がります！

#### 🛡️ 対策②：「許可しますか？」の画面をちゃんと読む

Claude Code が何かを実行するとき、こんな感じで **「許可しますか？」** と聞いてくる画面が出ることがあります。

![](../assets/do-you-want-to-proceed.png)

ここで大事なポイントは **2つ** です。

1. **中身を読まずに「Yes」を連打しない！**
   - 何をしようとしているか、ざっくりでいいので確認しましょう
   - よく分からないコマンドが出てきたら、 **Claude Code に「これは何をするコマンド？」と聞いてもOK** です
2. **「Always allow（常に許可）」は慎重に！**
   - 「Yes」は今回だけ許可、「Always allow」は今後ずっと許可という意味です
   - 慣れないうちは **毎回「Yes」で1つずつ確認** するのがおすすめです

#### 🛡️ 対策③：サンドボックスモードを設定する

Claude Code を起動した状態で、次のコマンドを実行しましょう。

```
/sandbox
```

**サンドボックス** とは、Claude Code が触れるファイルやインターネット接続を制限する機能です。
「この範囲の中だけで作業してね」と、行動範囲を区切ってあげるイメージです。

モードを選択します。

```
※ どちらか好みの方を選択してください
  1. auto-allow モード: 安全なコマンドは自動で実行してOK
  2. regular モード: 安全なコマンドでも毎回確認してから実行
```

> 💡 初心者の方は **regular モード** がおすすめです。毎回確認が出るので安心です。

#### 🛡️ （豆知識）秘密の情報をファイルに書いたときは要注意

今日のハンズオンでは使いませんが、今後 Claude Code を使っていくと、APIキー（サービスを使うための秘密の鍵）やパスワードなどを `.env` というファイルに書くことがあるかもしれません。

このファイルをうっかり **GitHub（みんなが見れる場所）にアップロード** してしまうと、秘密の情報が全世界に公開されてしまいます。

> 💡 `.gitignore` という仕組みを使うと「このファイルはアップロードしないでね」と指定できます。
> もし今後 `.env` を使うことがあったら、 **Claude Code に「.env を .gitignore に追加して」と頼む** だけでOKです！覚えておくと安心です。

> もしものときのコマンド
> esc キー →前の画面に戻る
> ctrl+c  →操作を中断する
> /resume →前のチャットの履歴から再開する
> /login  →ログイン
> /logout →ログアウト

### Claude Code を使うときのコツ

Claude Code をもっと上手に使うためのコツを紹介します。

  - ❌「いい感じにして」 → Claude Code は何をすればいいか分かりません

  - 何度でもやり直せるのが Claude Code の良いところです
- **困ったら Claude Code 自身に聞く**
  - 「このエラーはどういう意味？」「次は何をすればいい？」と聞けば、Claude Code が教えてくれます

## Claude デスクトップアプリで「コード」を試してみましょう

ここまではターミナルで Claude Code を使ってきましたが、実は **Claude のデスクトップアプリ** でも同じようなことができます。

1. デスクトップアプリを起動する
2. アプリ上部タブにある **「コード」** をクリック
3. 作業フォルダを指定
4. あとはターミナルのClaude Codeと同じように指示を出すだけ！

ターミナルとの違いは、**チャット画面の中でコードの実行結果やプレビューが見られる** ことです。
コマンド操作が苦手な方は、こちらの方が直感的に使えるかもしれません。

## Claude デスクトップアプリで「Cowork」を試してみましょう

**Cowork** は、Claude Code のさらに進化版のような機能です。

Claude Code との違いは、**途中の確認なしに、最初から最後まで全自動でやり切ってくれる** ことです。

1. デスクトップアプリを起動する
2. アプリ上部タブにある **「Cowork」** をクリック
3. やってほしいことを指示する
4. あとは待つだけ！完成したら結果を報告してくれます

> 💡 例えば「○○のチラシをHTMLで作って」のように、成果物がはっきりしている指示が向いています。
>
> ⚠️ Cowork はまだ研究段階（research preview）の機能です。うまくいかないこともあるので、気軽に試してみましょう！
