# NotebookLM インフォグラフィック作成プロンプト: Claude Code が使える場所

## 使い方

1. NotebookLM でノートブックを開く
2. ソースとして `handson-guide.md` をアップロードする
3. **Studio → Infographic** に移動する
4. 以下のプロンプトを貼り付けて生成する

## 設定

- **向き**: ポートレート（縦長）— 3つの場所を縦に並べて比較
- **詳細度**: Concise（簡潔）— 内容がシンプルなので簡潔に
- **スタイル**: Instructional（使い分けガイドに最適）

## プロンプト

```
以下の内容をインフォグラフィックにしてください。

【テーマ】Claude Code はどこで使える？3つの場所を比較

【対象者】Claude Code をこれから使い始める初心者。技術用語は最小限に、親しみやすく。

【構成】ヘッダー、3つの比較ブロック、おすすめガイド、フッターの6ブロック構成。1画面に収めてください。

【ブロック内容】

1. ヒーローブロック（タイトル）
   - タイトル: 「Claude Code 3つの使い方」
   - サブタイトル: 自分に合った場所を選ぼう

2. 場所①: ターミナル（黒い画面）
   - アイコン: 🖥️ ターミナル画面
   - ニックネーム: 「通称：黒い画面」
   - 良いポイント: 最新機能がいち早く使える！
   - デメリット: コマンド操作に慣れが必要
   - おすすめ度: ★★★（上級者向け）

3. 場所②: VSCode（高性能メモ帳）
   - アイコン: 📝 エディタ画面
   - ニックネーム: 「エンジニア御用達の高性能メモ帳」
   - 良いポイント: 変更ファイルが確認しやすい
   - デメリット: ターミナル版より機能が限定的なことも
   - おすすめ度: ★★☆（中級者向け）

4. 場所③: デスクトップアプリ
   - アイコン: 💬 チャットアプリ
   - ニックネーム: 「Claude公式アプリ」
   - 良いポイント: チャットで慣れ親しんだデザイン
   - デメリット: ターミナル版より機能が限定的なことも
   - おすすめ度: ★☆☆（初心者向け）

5. おすすめガイド
   - 初心者: 「まずはアプリから始めよう！」
   - 慣れてきたら: 「VSCodeで効率アップ！」
   - もっと使いこなす: 「ターミナルで最新機能を体験！」
   - ステップアップの矢印で段階を表現してください

6. フッター
   - 「今日のハンズオンではターミナルに挑戦！」

【デザイン指示】
- カラーパレット: メイン #D97757（Claudeブランドオレンジ）、サブ #1A1A2E（ダークネイビー）、アクセント #F5F0EB（ライトベージュ）
- 3つの場所はカード形式で、メリット/デメリットが一目で比較できるようにしてください
- 各カードに良いポイントは緑系、デメリットはグレー系の色分けで視覚的に区別
- タイトルは8語以内、ヘッダーは5語以内、箇条書きは10語以内
- ホワイトスペースを十分に確保し、読みやすさを最優先
- 装飾は最小限。アイコンで各場所を直感的に区別してください
```
